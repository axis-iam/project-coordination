项目协调

把项目决策、计划、任务、代码质量和验证证据，整理成下一项可验收的工作。

安装到项目 查看源码
协调契约 先定范围，再写代码。
先有证据，再做验收。

从真实的多仓库交付流程中提炼，不绑定具体产品或技术栈。

五个职责清晰的 Skill

从当前工作线，一直到可验证的交付证据。

Workstream Triage

用决策、计划、任务索引和仓库证据确认主工作线，区分阻塞项、未来工作与待决策事项。

Project Coordination

把已确认方向转成计划与可执行任务，交给新 session 实现，再回到协调 session 独立验收。

Validation Harness

修改前定义最小可复现验证，区分工具、运行时与端到端证据，并按 lane 组织 live smoke。

API Contract Sync

确认权威契约来源，同步后端、前端、Mock、生成客户端、文档和 SDK 的结构与行为。

Code Quality Audit

聚焦 changed files，找出过度兜底、异常掩盖、宽松类型和不必要状态同步等审查候选。

核心规则

分离协调、执行与验收。

01

协调

创建权威任务，解决依赖，分配 worktree，声明验证 harness，并生成交接 prompt。

02

在新 session 中执行

只实现声明范围，运行 harness，回写执行记录。不暗中扩大范围，也不自行验收。

03

独立验收

回到协调 session，检查 diff、证据、依赖和消费者契约后，再记录验收结果。

按规模启用流程

只使用工作真正需要的流程。

安装时选择项目默认档位；每个 tracked task 根据实际风险记录有效档位，可升级也可降级。

Compact

局部、低风险改动可以留在一个 session 中，并配合聚焦的验证命令。

Standard

后端与前端协作使用任务记录、契约负责人、新执行 session 和协调验收。

Complex

多仓库、SDK、并行工作、迁移和外部 gate 使用明确的依赖与发布控制。

随协调包安装

审查可疑模式，不用扫描结果代替判断。

聚焦变更

按真实 Git 根检查 changed files，支持 Java、Kotlin、TypeScript、JavaScript、Go 和 Python。

人工分类

识别过度兜底、异常吞噬、空结果掩盖、any、浏览器 token 存储和 React 状态同步候选。

证据分离

扫描结果只是 review 输入；编译、测试、运行时和 E2E 证据仍由执行与验收记录负责。
